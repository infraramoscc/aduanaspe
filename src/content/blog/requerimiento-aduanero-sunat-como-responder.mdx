## Qué es un requerimiento aduanero de SUNAT

Un **requerimiento aduanero** es una solicitud formal de información, documentos, explicaciones o sustento vinculados a una operación de comercio exterior. Puede aparecer durante el despacho, después del despacho, dentro de una fiscalización o como parte de una actuación de control.

Responderlo mal puede generar problemas adicionales:

- que SUNAT considere que no atendiste lo solicitado
- que se mantenga una observación
- que el caso derive en multa o determinación tributaria
- que se presenten documentos contradictorios
- que pierdas la oportunidad de explicar el caso con orden

La clave es no tratar el requerimiento como una carga administrativa más. Es una oportunidad para ordenar la operación y responder al punto exacto.

## Paso 1: entiende qué te están pidiendo

Antes de buscar documentos, lee el requerimiento y separa:

| Pregunta | Por qué importa |
|---|---|
| ¿Qué operación está observada? | Permite ubicar DAM, factura, transporte y expediente correcto |
| ¿Qué documento o información solicita SUNAT? | Evita responder con archivos que no contestan la pregunta |
| ¿Qué plazo tienes? | Determina la urgencia y si necesitas pedir documentos hoy |
| ¿Qué supuesto se está investigando? | Cambia la estrategia: valor, tributos, permiso, factura, origen, régimen |
| ¿Qué consecuencia menciona? | Ayuda a medir si hay riesgo de multa, ajuste o fiscalización más amplia |

<Callout type="warning" title="No respondas por intuición">
  Si el requerimiento menciona un artículo, código, supuesto o régimen, revisa su significado antes de escribir. Una respuesta genérica puede dejar preguntas sin contestar.
</Callout>

## Paso 2: arma una matriz de documentos

Una matriz simple puede evitar errores. Usa tres columnas:

| Documento | Estado | Observación |
|---|---|---|
| Factura comercial | Disponible / falta / inconsistente | Revisar si coincide con DAM |
| Packing list | Disponible / falta / inconsistente | Ver peso, bultos, descripción |
| Documento de transporte | Disponible / falta / inconsistente | Revisar datos de consignatario y mercancía |
| Medio de pago | Disponible / falta / inconsistente | Comparar monto, proveedor y fecha |
| Permiso o autorización | Disponible / no aplica / falta | Validar si el producto era restringido |
| Comunicaciones | Disponible / falta | Sirve para explicar precio, descuento o condición |

Esta matriz no reemplaza el expediente, pero te muestra rápidamente si estás listo para responder o si debes pedir soporte antes de enviar algo.

## Paso 3: identifica el tipo de riesgo

Los requerimientos aduaneros suelen caer en alguno de estos grupos:

### Riesgo de valor

SUNAT quiere entender si el valor declarado se sustenta. Revisa factura, pagos, descuentos, flete, seguro, condiciones de venta y documentos comerciales. Para este caso, usa como referencia [ajuste de valor SUNAT](/blog/ajuste-de-valor-sunat-importacion/) y [servicio de ajuste de valor aduanero](/servicios/ajuste-de-valor-aduanero/).

### Riesgo documental

La preocupación está en documentos incompletos, con errores, no veraces, no auténticos o transmitidos de forma incorrecta. Este riesgo puede conectar con [multas aduaneras SUNAT](/blog/multas-aduaneras-sunat-como-evitarlas/).

### Riesgo regulatorio

El producto podría ser restringido, requerir permiso, autorización, visto bueno o certificado. Revisa [permisos para importar en Perú](/blog/como-saber-si-tu-producto-necesita-permisos-para-importar-en-peru/).

### Riesgo tributario

El problema impacta tributos, recargos, base imponible, percepción, clasificación o aplicación de una preferencia. Revisa también [tributos de importación en Perú](/blog/calcular-tributos-importacion-peru/).

## Paso 4: responde con estructura

Una respuesta ordenada suele contener:

1. Identificación del requerimiento.
2. Resumen breve de la operación.
3. Punto observado por SUNAT.
4. Explicación técnica o documental.
5. Relación de documentos que sustentan la respuesta.
6. Anexos ordenados y nombrados.
7. Cierre indicando que se atiende lo solicitado.

No es necesario escribir demasiado si el caso es simple. Pero si el requerimiento toca valor, permisos, infracción o fiscalización, conviene que la respuesta tenga una lógica clara.

## Errores que debes evitar

- Enviar documentos sin índice.
- Adjuntar archivos ilegibles.
- Responder algo distinto a lo solicitado.
- Presentar factura, packing list y DAM con datos que no coinciden.
- Ignorar la fecha de notificación.
- Asumir que el agente, proveedor o operador ya reviso todo.
- No guardar constancia de lo presentado.

## Cuándo pedir revisión técnica

Pide apoyo si:

- el requerimiento menciona infracción, sanción o multa
- el plazo está cerca
- hay inconsistencias entre documentos
- el valor declarado está siendo observado
- la mercancía requiere permisos o autorizaciones
- la operación ya fue despachada y ahora entra en fiscalización posterior
- no sabes si responder, subsanar o preparar descargo

En esos casos, una revisión previa puede evitar que respondas demasiado, demasiado poco o con documentos que abren otra contingencia.
