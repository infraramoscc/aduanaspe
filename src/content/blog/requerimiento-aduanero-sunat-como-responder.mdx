## Que es un requerimiento aduanero de SUNAT

Un **requerimiento aduanero** es una solicitud formal de informacion, documentos, explicaciones o sustento vinculados a una operacion de comercio exterior. Puede aparecer durante el despacho, despues del despacho, dentro de una fiscalizacion o como parte de una actuacion de control.

Responderlo mal puede generar problemas adicionales:

- que SUNAT considere que no atendiste lo solicitado
- que se mantenga una observacion
- que el caso derive en multa o determinacion tributaria
- que se presenten documentos contradictorios
- que pierdas la oportunidad de explicar el caso con orden

La clave es no tratar el requerimiento como una carga administrativa mas. Es una oportunidad para ordenar la operacion y responder al punto exacto.

## Paso 1: entiende que te estan pidiendo

Antes de buscar documentos, lee el requerimiento y separa:

| Pregunta | Por que importa |
|---|---|
| Que operacion esta observada? | Permite ubicar DAM, factura, transporte y expediente correcto |
| Que documento o informacion solicita SUNAT? | Evita responder con archivos que no contestan la pregunta |
| Que plazo tienes? | Determina la urgencia y si necesitas pedir documentos hoy |
| Que supuesto se esta investigando? | Cambia la estrategia: valor, tributos, permiso, factura, origen, regimen |
| Que consecuencia menciona? | Ayuda a medir si hay riesgo de multa, ajuste o fiscalizacion mas amplia |

<Callout type="warning" title="No respondas por intuicion">
  Si el requerimiento menciona un articulo, codigo, supuesto o regimen, revisa su significado antes de escribir. Una respuesta generica puede dejar preguntas sin contestar.
</Callout>

## Paso 2: arma una matriz de documentos

Una matriz simple puede evitar errores. Usa tres columnas:

| Documento | Estado | Observacion |
|---|---|---|
| Factura comercial | Disponible / falta / inconsistente | Revisar si coincide con DAM |
| Packing list | Disponible / falta / inconsistente | Ver peso, bultos, descripcion |
| Documento de transporte | Disponible / falta / inconsistente | Revisar datos de consignatario y mercancia |
| Medio de pago | Disponible / falta / inconsistente | Comparar monto, proveedor y fecha |
| Permiso o autorizacion | Disponible / no aplica / falta | Validar si el producto era restringido |
| Comunicaciones | Disponible / falta | Sirve para explicar precio, descuento o condicion |

Esta matriz no reemplaza el expediente, pero te muestra rapidamente si estas listo para responder o si debes pedir soporte antes de enviar algo.

## Paso 3: identifica el tipo de riesgo

Los requerimientos aduaneros suelen caer en alguno de estos grupos:

### Riesgo de valor

SUNAT quiere entender si el valor declarado se sustenta. Revisa factura, pagos, descuentos, flete, seguro, condiciones de venta y documentos comerciales. Para este caso, usa como referencia [ajuste de valor SUNAT](/blog/ajuste-de-valor-sunat-importacion/) y [servicio de ajuste de valor aduanero](/servicios/ajuste-de-valor-aduanero/).

### Riesgo documental

La preocupacion esta en documentos incompletos, con errores, no veraces, no autenticos o transmitidos de forma incorrecta. Este riesgo puede conectar con [multas aduaneras SUNAT](/blog/multas-aduaneras-sunat-como-evitarlas/).

### Riesgo regulatorio

El producto podria ser restringido, requerir permiso, autorizacion, visto bueno o certificado. Revisa [permisos para importar en Peru](/blog/como-saber-si-tu-producto-necesita-permisos-para-importar-en-peru/).

### Riesgo tributario

El problema impacta tributos, recargos, base imponible, percepcion, clasificacion o aplicacion de una preferencia. Revisa tambien [tributos de importacion en Peru](/blog/calcular-tributos-importacion-peru/).

## Paso 4: responde con estructura

Una respuesta ordenada suele contener:

1. Identificacion del requerimiento.
2. Resumen breve de la operacion.
3. Punto observado por SUNAT.
4. Explicacion tecnica o documental.
5. Relacion de documentos que sustentan la respuesta.
6. Anexos ordenados y nombrados.
7. Cierre indicando que se atiende lo solicitado.

No es necesario escribir demasiado si el caso es simple. Pero si el requerimiento toca valor, permisos, infraccion o fiscalizacion, conviene que la respuesta tenga una logica clara.

## Errores que debes evitar

- Enviar documentos sin indice.
- Adjuntar archivos ilegibles.
- Responder algo distinto a lo solicitado.
- Presentar factura, packing list y DAM con datos que no coinciden.
- Ignorar la fecha de notificacion.
- Asumir que el agente, proveedor o operador ya reviso todo.
- No guardar constancia de lo presentado.

## Cuando pedir revision tecnica

Pide apoyo si:

- el requerimiento menciona infraccion, sancion o multa
- el plazo esta cerca
- hay inconsistencias entre documentos
- el valor declarado esta siendo observado
- la mercancia requiere permisos o autorizaciones
- la operacion ya fue despachada y ahora entra en fiscalizacion posterior
- no sabes si responder, subsanar o preparar descargo

En esos casos, una revision previa puede evitar que respondas demasiado, demasiado poco o con documentos que abren otra contingencia.
